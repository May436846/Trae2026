一、平台概述

微宏BPA流程规划设计平台是一款企业级业务流程架构（Business Process Architecture）管理与设计平台，帮助企业完成从流程规划、要素定义、文件编制、资料归档到监控运营的全生命周期管理。平台基于"流程架构树"组织企业全部业务流程，并通过管理要素（角色、风险、术语、指标等）与流程节点关联，实现流程与体系文件的联动管理。

二、平台双端架构

平台采用"门户浏览 + 工作台管理"双端架构，分别面向不同用户群体：

个人门户端（面向最终用户）

入口：登录后默认进入，或点击"返回门户"

  • 我的主页 — 个人工作台与待办
  • 流程 — 业务流程架构浏览
  • 制度 — 制度文件查阅
  • 文件库 — 知识文档管理
  • 组织机构 — 组织架构与责任查询
  • 管理要素 — 风险/控制/术语/指标/角色浏览
  • 统计分析 — 体系数据看板

管理后台端（面向管理员/设计者）

入口：点击顶部"进入工作台"按钮

  • 我的工作台 — 任务中心
  • 流程建设 — 流程架构设计
  • 要素定义 — 管理要素维护
  • 文件编制 — 体系文件编写
  • 资料归档 — 文档归档与作废
  • 监控运营 — 运行监控与消息
  • 统计分析 — 数据分析
  • 系统设置 — 系统配置管理

2. 流程建设 流程架构设计（核心模块）

二级模块三级模块功能说明交互流程
新建节点
（三种类型）		 新建根节点创建流程架构的起始节点（一般为公司名称），系统默认根节点为L0层级。支持新建多流程架构根节点，每个根节点下的架构与流程可独立管理维护（应对多管理架构情况）
  1. 点击"流程建设"进入
  2. 左上角点击"新建"→"新建根节点"
  3. 输入根节点名称
  4. 鼠标移开点击空白处完成创建
新建架构在根节点下创建架构细分，用于描述流程的分级和分类，展示流程分布情况，可根据业务细分程度建立多层子级架构（L1、L2、L3...）
  1. 选中父节点
  2. 点击"新建"→"新建架构"
  3. 输入架构名称
  4. 系统自动生成编号
  5. 鼠标移开完成创建
新建流程在架构层级下创建具体流程，由若干个连续的活动组成，用于描述各岗位角色人员在业务各环节中的工作内容（L4及以下为流程级）
  1. 选中架构节点
  2. 点击"新建"→"新建流程"
  3. 输入流程名称
  4. 系统自动生成编号
架构搭建
方式		 手工搭建通过"新建根节点→新建架构→新建流程"逐级手工创建，选中节点后再点击对应新建类型即可逐层构建
  1. 新建根节点(L0)
  2. 选中根节点→新建架构(L1)
  3. 选中L1→新建架构(L2)...
  4. 选中架构→新建流程(L4)
Excel批量导入下载导入模版（架构模版、流程模版分开），按模版整理路径、层级、名称等信息后批量导入，系统自动生成分层分级流程架构树。注意：架构和流程需用两个模版分别导入，一般先导入架构再导入流程
  1. 先创建一个根节点
  2. 下载《架构模版》
  3. 从第三行起录入路径、层级、名称
  4. 责任人/部门需系统已有数据
  5. 执行"架构导入"
  6. 再下载《流程模板》整理
  7. 执行"流程导入"
架构设计
（7个维度）		 架构图展现当前架构下属一级子节点的情况，通过组件库拖拽绘制。先拖基础图形，再拖节点组件，双击节点组件关联架构树上对应层级节点，支持下钻跳转查看
  1. 选中架构节点
  2. 切换"架构图"标签
  3. 点击"全屏"放大画布
  4. 拖拽基础图形
  5. 拖拽节点组件置于其上
  6. 双击节点组件关联架构节点
  7. 支持下钻跳转
架构视图业务全景图/集成关系图，展现当前架构下各子业务之间、流程之间的业务逻辑关系，描述业务全过程
  1. 切换"架构视图"标签
  2. 全屏绘制
  3. 拖拽组件绘制业务全景

五、核心交互流程

流程0A：流程架构搭建（手工方式）

步骤操作说明
1进入流程建设工作台→流程建设，左上角点击"新建"
2新建根节点(L0)选择"新建根节点"，输入公司名称（如"阿尔法集团"），鼠标移开完成创建
3新建架构(L1)选中根节点→"新建"→"新建架构"，输入架构名称（如"1.0管理战略DSTE"），系统自动生成编号
4逐层新建架构(L2/L3)选中L1→新建架构(L2)，选中L2→新建架构(L3)，根据业务细分程度建立多层子级架构
5新建流程(L4)选中L3架构节点→"新建"→"新建流程"，输入流程名称（如"1.1.1战略方向"），系统自动生成编号
6架构设计选中节点，切换"架构图/架构视图/架构卡"等标签进行设计
7关联要素与文件切换"相关风险/相关指标/相关管理手册"等标签关联数据
8发布选择"直接发布"或"审批发布"使架构生效

流程0B：流程架构搭建（Excel批量导入）

步骤操作说明
1创建根节点先手工新建一个根节点作为导入基础
2下载架构模版点击"导入"→下载《架构模版》Excel文件
3整理架构数据从第三行起录入：路径、层级、名称；责任人/部门需为系统已有数据
4执行架构导入上传整理好的Excel文件，系统自动生成分层分级架构树
5下载流程模版下载《流程模版》Excel文件
6整理流程数据从第三行起录入：路径、层级、名称等信息
7执行流程导入上传流程Excel文件，系统自动在架构树下生成流程节点

流程0C：流程设计完整流程

步骤操作说明
1进入流程设计工作台→流程建设→选中流程节点→点击"流程设计"
2全屏绘制点击"全屏"放大画布
3拖拽泳道图选择水平/垂直/阶段泳道，拖至画布
4关联角色双击泳道，从角色库选择角色（或直接创建新角色）
5关联岗位双击泳道，从岗位库关联岗位（建议关联"客户体验"岗以便体验消息通知）
6拖拽建模组件拖拽起始组件→活动组件→决策组件，构建流程图
7设计活动属性双击活动组件，填写名称（编号自动生成）、描述，关联文件/术语/风险/指标
8（可选）Visio导入如有Visio流程图，点击"导入"→"Visio导入"，建立组件转换关系，导入后"自动编号"
9维护基本属性切换"基本属性"页签，填写必填项（红色星号），选择责任人/部门
10关联要素文件切换"相关文件/术语/风险/指标"等标签关联数据
11检查文件内容切换"文件内容"页签查看自动同步的结构化信息
12发布流程选择"直接发布"或"审批发布"；发布后自动生成存档文件，并推送消息和学习任务给关联岗位人员

流程1：流程架构浏览与钻取

步骤操作说明
1登录平台输入账号密码，进入个人门户首页
2点击顶部"流程"进入流程模块，左侧显示业务流程架构树
3展开架构节点点击L1/L2节点（如"1.0管理战略DSTE"）展开下级流程
4选择流程节点点击L3/L4节点（如"1.1制定战略规划SP"），右侧加载视图
5切换视图标签在架构卡/键盘图/架构图等13个标签页间切换
6查看关联要素切换"相关风险/相关指标/相关术语"等标签查看关联要素
7导出/反馈点击右上角"导出"或"意见反馈"

流程2：制度文件查阅

步骤操作说明
1点击顶部"制度"进入制度模块
2选择分类左侧制度分类树选择分类（如"营销管理"）
3搜索制度搜索框输入制度名称搜索
4查看制度点击列表"查看"操作，查看制度详情

流程3：工作台文件编制与发布

步骤操作说明
1进入工作台点击顶部"进入工作台"按钮
2选择文件类型左侧"文件编制"展开，选择制度管理/管理手册/指导书/规范标准
3创建文件新增文件，编辑内容，关联流程节点
4提交审批提交发布审批，进入审批流程
5审批处理审批人在"我的工作台→编制任务"或BPMA流程→待办流程中处理
6归档发布审批通过后文件发布，自动归档至资料归档→文件库

流程4：流程宣贯与学习

步骤操作说明
1发起宣贯组织机构模块选择部门，点击"流程宣贯"发起
2配置学习任务工作台→监控运营→学习任务配置，创建学习任务
3用户学习用户在门户"我的主页→待学习"中查看并学习
4学习完成完成后状态变为"已学习"

流程5：审批流程设计（BPMA）

步骤操作说明
1进入流程设计工作台→系统设置→BPMA流程→流程设计
2设计审批流在画布中拖拽节点，设计审批流程图
3配置节点为每个审批节点配置审批人、条件、规则
4保存发布保存流程设计并发布
5启动流程在BPMA流程→启动流程中发起审批实例
6监控运行在流程监控中查看运行状态，处理超时
